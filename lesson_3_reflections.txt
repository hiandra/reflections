Quando gostaria de usar um repositório remoto em vez de manter o trabalho local?
Usar um repositório remoto é mais interessante do que manter o trabalho local quando são feitas alterações pelo mesmo usuário através de diferentes computadores e quando há mais de um colaborador no projeto, de forma que podem ser feitas as alterações de forma local e atualizar o projeto no repositório remoto após realizá-las.

Por que você prefere efetuar pull de alterações manualmente do que deixar o Git atualizar-se automaticamente com o seu repositório remoto?
É preferível efetuar pull de alterações manualmente do que deixar o Git atualizar-se automaticamente com o repositório remoto porque com sincronização automática pode ser que sejam tornadas públicas features que não estão totalmente prontas, já que geralmente o desenvolvimento de uma nova feature ocorre após vários commits.

Descreva as diferenças entre bifurcações, clones e branches. Quando você usaria um e não os outros?
Bifurcações (Forks) são cópias de repositórios de outras pessoas diretamente nos servidores do Github sem baixar o código para a máquina local. Essa cópia aparecerá diretamente na conta de quem copiou o repositório. Já clones consitem em copiar o repositório dos servidores do Github para a máquina local e branches consitem em subdivisões do histórico de commits de forma que esses se ramficam em múltiplas versões, sendo cada uma delas um branch. Bifurcações são usadas quando se deseja copiar o repositório de um usuário para fazer alterações de forma a dar crédito ao autor da versão original, visto que aparecerá um link no repositório bifurcado para o original. Já os clones são usados quando se deseja alterar os arquivos de um repositório localizado no servidor do Github na máquina local, visto que, usando o clone, estes serão copiados para ela. Finalmente, os branches são usados quando se deseja iniciar o desenvolvimento de uma nova feature sem interfeir no código que já está funcionando da linha de desenvolvimento principal.

Qual é o benefício de ter uma cópia do último estado conhecido do remoto armazenado localmente?
O benefício de ter uma cópia do última estado conhecido do remoto armazanado localmente é que se o desenvolvedor for ficar offline por um tempo poderá atualizar a cópia local logo antes de ficar offline e depois terá acesso a qualquer atualização que tenha sido feita no remoto desde que atualizou pela última vez. Além disso, após realizar a cópia pode-se usar os comandos git log e git diff para verificar as diferenças entre os branchs remoto e local, o que facilita um merge posterior.

Como você colaboraria sem usar Git ou GitHub? O que seria mais fácil e mais difícil?
Eu colaboraria sem usar Git ou Github usando, por exemplo, Dropbox. Seria mais difícil acompanhar as novas atualizações dos arquivos, como verificar as mudanças e evitar perdas de versões sobrescrevendo arquivos. Seria mais fácil a parte inicial do uso, que com o Git é necessário aprender o seu funcionamento, já o droppbox não oferece tantas funcionalidades, deixando-o intuitivo.
